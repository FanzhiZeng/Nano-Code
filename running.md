# RUNNING

## 当前目标

把当前原型先整理成“稳定可跑的聊天 CLI 基线”，再开始接工具调用。当前 `main.py` 已经能发请求，但配置校验、退出体验和异常处理都还比较脆弱。

## 本轮范围

只处理基础运行链路，不进入工具调用实现。

包括：

- 配置加载与校验
- 客户端初始化安全性
- 对话循环的退出与异常处理
- 输出更清晰的启动信息

不包括：

- 文件工具
- shell 工具
- 自动扫描项目上下文
- 多模块重构

## 具体事项

### 1. 修正启动配置逻辑

当前问题：

- `api_key` 只在 `ANTHROPIC_BASE_URL` 存在时才会赋值
- 如果 `ANTHROPIC_BASE_URL` 为空，`client = Anthropic(...)` 可能引用未定义变量
- `MODEL_ID` 缺失时只打印 `Model not found!`，提示太弱

要做的事：

- 单独读取 `ANTHROPIC_API_KEY`
- 单独读取 `ANTHROPIC_BASE_URL`
- 单独读取 `MODEL_ID`
- 对必填项做显式校验
- 报错时指出缺少哪个环境变量

建议落点：

- [main.py](/home/luo/project/nano_code/main.py) 启动部分

### 2. 统一客户端初始化

当前问题：

- 初始化逻辑写在模块顶层，导入即执行，不利于后续扩展
- `base_url` 是否为空没有单独处理

要做的事：

- 增加一个初始化函数，例如 `build_client()`
- 当 `ANTHROPIC_BASE_URL` 为空时，直接走默认 Anthropic 地址
- 当 `ANTHROPIC_BASE_URL` 有值时，再显式传入

验收点：

- `.env` 里不填写 `ANTHROPIC_BASE_URL` 也能启动
- 填写兼容网关地址时也能启动

### 3. 补齐 REPL 退出体验

当前问题：

- `input()` 遇到 `Ctrl+D` 会抛 `EOFError`
- `Ctrl+C` 中断时缺少友好退出信息

要做的事：

- 捕获 `EOFError`
- 捕获 `KeyboardInterrupt`
- 退出前打印简短提示，例如 `Bye.`

验收点：

- `Ctrl+C` 能正常结束进程
- `Ctrl+D` 能正常结束进程
- 不出现长栈追踪

### 4. 补充请求异常处理

当前问题：

- `client.messages.create(...)` 失败时会直接抛异常

要做的事：

- 给单次请求加 `try/except`
- 请求失败时打印错误摘要，不丢失会话状态
- 用户可以继续下一轮输入，而不是程序直接退出

验收点：

- 模拟错误配置时，程序能给出可理解提示
- 修正配置后可以继续重新运行

### 5. 明确本轮完成标准

完成这一步后，程序应满足：

- 配置错误时，报错信息可读
- 配置正确时，程序稳定进入聊天循环
- 用户可以通过 `Ctrl+C` 或 `Ctrl+D` 退出
- 网络或鉴权失败时，不出现无法理解的崩溃输出

## 建议实施顺序

1. 先整理环境变量读取和校验
2. 再封装客户端初始化
3. 然后处理 REPL 退出逻辑
4. 最后补请求异常处理

## 手动验证

建议至少做下面 4 组检查：

1. 删除 `MODEL_ID` 后启动，确认提示清楚
2. 清空 `ANTHROPIC_BASE_URL` 后启动，确认仍可初始化客户端
3. 启动后按 `Ctrl+C`，确认正常退出
4. 故意填错 `ANTHROPIC_API_KEY`，确认请求失败时输出可读错误
